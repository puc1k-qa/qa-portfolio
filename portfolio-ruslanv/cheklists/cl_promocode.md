# Чек-лист тестирования функционала (Промокоды)

### Позитивные сценарии
- [ ] **Валидный код** — успешная активация и сообщение «Промокод применён».
- [ ] **Пересчет суммы** — корректное уменьшение итоговой цены заказа.
- [ ] **Сводка заказа** — отображение скидки отдельной строкой в деталях платежа.

### Негативные сценарии и валидация
- [ ] **Несуществующий код** — вывод ошибки «Промокод не найден».
- [ ] **Состояние корзины** — сумма заказа не меняется при ошибке.
- [ ] **Минимальный порог** — ошибка при сумме заказа ниже условий акции.
- [ ] **Истекший срок** — проверка обработки просроченных промокодов.
- [ ] **Повторное использование** — блокировка кода, если он уже был применен ранее.

### Интерфейс и UX
- [ ] **Регистр** — работа кода вне зависимости от больших/маленьких букв (Caps Lock).
- [ ] **Пробелы** — автоматическая очистка пробелов по краям (trim).
- [ ] **UI Ошибок** — подсветка полей и текста ошибок красным цветом.
- [ ] **Удаление** — возможность отменить примененный промокод.
